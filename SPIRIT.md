# Ladingburger (Saborea) — Spirit

## The vision in one sentence

A customer places an order in under 60 seconds. An admin processes it without confusion.

## What that means

Speed and clarity are the only two things that matter. The customer menu is built for someone who has never used the app before. The admin panel is built for someone who is stressed, busy, and handling 10 orders at once.

## The app's personality

- Fast. Every interaction should feel instant.
- Clear. Status, actions, and information are never ambiguous.
- Opinionated. The app decides the flow — the user follows it, not the other way around.

## Core beliefs

- A great ordering experience removes all friction between "I'm hungry" and "order confirmed."
- Kitchen staff and admin should never need training to use this app.
- Complexity belongs in the code, not in the interface.

## Anti-patterns — things this app never does

- Adds steps to the ordering flow without a strong reason.
- Shows the customer information they don't need to place an order.
- Builds an admin feature that requires explanation or a tutorial.
- Adds options that slow down the most common action (place order / update status).

## Feature alignment test

Before building any new feature, ask:
1. Does this make the customer's order faster or clearer?
2. Does this help admin process orders without confusion?
3. Does it fit into the existing flow without adding a new concept?

If the answer to any is "no," don't build it.

## What the app is NOT

- Not a full restaurant management system.
- Not a POS system.
- Not a marketing or loyalty platform.
- Not a tool that requires an onboarding tutorial.
